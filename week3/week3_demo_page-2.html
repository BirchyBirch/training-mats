CSS Advanced Selectors & Modern Properties - Week 3

1. Logical vs Physical Properties

Modern CSS uses logical properties that adapt to different writing directions and languages.

/* Physical Properties (old way) */ .old-card { margin-top: 16px; margin-right: 20px; margin-bottom: 16px; margin-left: 20px; padding-left: 24px; border-left: 4px solid blue; } /* Logical Properties (modern way) */ .new-card { margin-block: 16px; /* top and bottom */ margin-inline: 20px; /* left and right */ padding-inline-start: 24px; /* left in LTR, right in RTL */ border-inline-start: 4px solid blue; }
Physical Properties
Fixed to physical directions (top, right, bottom, left)
Logical Properties
Adapts to writing direction and language

2. Pseudo-elements in Action

Pseudo-elements let you style specific parts of elements or create virtual elements.

/* ::before and ::after */ .quote::before { content: """; } .quote::after { content: """; } /* ::first-letter for drop caps */ .article::first-letter { font-size: 3em; float: left; } /* ::first-line emphasis */ .paragraph::first-line { font-weight: bold; }
This is a quote with decorative quotation marks added using ::before and ::after pseudo-elements.

3. Interactive Pseudo-classes

Pseudo-classes provide enhanced interactivity and form validation.

/* Interactive states */ .button:hover { transform: translateY(-2px); } .button:focus { outline: 2px solid blue; } .button:active { transform: translateY(0); } /* Form validation */ input:valid { border-color: green; } input:invalid { border-color: red; } input:required::after { content: " *"; }

4. Structural Pseudo-classes

Target elements based on their position or relationship to other elements.

/* Position-based */ li:first-child { background: lightblue; } li:last-child { background: lightyellow; } li:nth-child(even) { background: lightgray; } li:nth-child(3n) { color: blue; font-weight: bold; } /* Form and content states */ input:focus, button:hover { /* styles */ }

5. Key Takeaways

  • Logical Properties: Use margin-block, padding-inline for international sites
  • Pseudo-elements: Add visual elements without extra HTML using ::before/::after
  • Interactive States: Enhance UX with :hover, :focus, :active pseudo-classes
  • Form Validation: Style :valid, :invalid, :required states for better feedback
  • Structural Selectors: Use :nth-child() patterns for consistent styling
  • Modern CSS: Combine these techniques for cleaner, more maintainable code
