1. CSS Combinators - Targeting Related Elements

Building on Week 1's selectors, combinators let you target elements based on their relationship to other elements. This is essential for component styling in Angular.

/* Descendant combinator (space) - any nested element */ .card p { color: gray; } /* Child combinator (>) - direct children only */ .card > .header { font-weight: bold; } /* Adjacent sibling (+) - immediately following sibling */ h2 + p { margin-top: 0; } /* General sibling (~) - any following sibling */ h2 ~ p { font-size: 0.9rem; }

üÖ∞Ô∏è Angular Component Combinators

/* component.scss - scoped to this component */ :host { display: block; } /* Target direct child buttons only */ :host > button { margin-right: 0.5rem; } /* Target any descendant with .error class */ :host .error { color: red; } /* Target Angular Material components within this component */ :host mat-form-field + mat-form-field { margin-top: 1rem; } /* Target elements after specific Angular components */ :host mat-card ~ .actions { margin-top: 1.5rem; }

ViewEncapsulation Context: These combinators work within the component's template. With ViewEncapsulation.Emulated, they won't affect other components, making them safe for component-specific styling.

üìö Combinator Specificity

Important: Combinators themselves don't add to specificity, but each selector in the combination does.

/* Specificity: 0,0,2,0 (two classes) */ .card .header { } /* Specificity: 0,0,2,0 (two classes, > doesn't count) */ .card > .header { } /* Specificity: 0,0,1,1 (one class + one element) */ .card p { } /* Specificity: 0,0,0,2 (two elements) */ h2 + p { }

2. Box-Sizing: Modern Standard

Modern web development uses box-sizing: border-box as the standard. This means width and height include content, padding, and border.

/* Modern standard - width includes padding and border */ .component { box-sizing: border-box; width: 300px; padding: 20px; border: 2px solid #ccc; /* Total width is exactly 300px */ }

üÖ∞Ô∏è Angular Application Setup

/* Global styles.css - set once for entire application */ *, *::before, *::after { box-sizing: border-box; } /* Angular Material components expect this behavior */ /* CDK Layout utilities are designed around border-box */

Why this matters: Angular Material, CDK Layout, and modern CSS frameworks all assume border-box sizing. Setting this globally ensures consistent behavior across your entire application.

Here are real-world examples of spacing issues you'll encounter in Angular component development:

‚ùå Inconsistent Spacing

Random spacing values make components look unprofessional
/* Bad: Random values */ .card { padding: 8px 12px 6px 15px; margin: 3px 0px 7px 2px; }

‚úÖ Consistent Spacing Scale

Consistent spacing creates visual harmony and professionalism
/* Good: Consistent scale */ .card { padding: 1rem; /* 16px */ margin: 0.5rem 0; /* 8px top/bottom */ }

Recommended Spacing Scale

üÖ∞Ô∏è Angular Component Architecture Considerations

/* component.scss - ViewEncapsulation.Emulated (default) */ :host { display: block; box-sizing: border-box; padding: var(--spacing-md); } /* Angular Material components use border-box */ /* Your custom components should match this behavior */ .content-wrapper { width: 100%; padding: 1rem; border: 1px solid #ccc; /* With border-box: predictable 100% width */ /* Without border-box: width becomes 100% + 2rem + 2px */ } /* CSS Custom Properties for consistent spacing */ :host { --component-padding: 1rem; --component-gap: 0.5rem; }

Angular Integration: Angular Material components assume border-box sizing. When building custom components, matching this behavior ensures consistent integration with Material components and predictable responsive behavior.

üÖ∞Ô∏è Angular Responsive Design & Box Model

/* component.scss */ :host { display: block; box-sizing: border-box; } /* Grid layout that breaks without border-box */ .grid-container { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 1rem; } .grid-item { padding: 1rem; border: 1px solid #ccc; /* Without border-box: items overflow grid container */ /* With border-box: perfect fit within grid constraints */ } /* Angular CDK Layout for responsive breakpoints */ .content { padding: 1rem; } @media (min-width: 768px) { .content { padding: 2rem; /* border-box ensures predictable sizing across breakpoints */ } }

CDK Integration: Angular CDK's Layout module provides breakpoint observers. When combined with border-box sizing, responsive components maintain predictable dimensions across all screen sizes without calculation overhead.

7. Key Takeaways & Best Practices

  • Always use border-box: Makes width/height calculations predictable
  • Consistent spacing scale: Use 4px, 8px, 16px, 24px, 32px values
  • DevTools box model: Your best friend for debugging layout issues
  • Angular components: Set box-sizing on :host for component consistency
  • Margin collapsing: Understand when vertical margins merge
  • rem for spacing: Scales consistently across different font sizes
  • Visual inspection: Always check the computed box model, not just your CSS
