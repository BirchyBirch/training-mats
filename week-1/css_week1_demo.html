1. CSS Rule Structure

Every CSS rule follows the same pattern:

/* selector { property: value; } */ h1 { color: blue; font-size: 24px; }

Parts of a CSS Rule:

  • Selector: What element(s) to style
  • Property: What aspect to change (color, size, etc.)
  • Value: How to change it
  • Declaration: property + value pair

2. Types of Selectors

/* Element Selector - targets all <p> tags */ p { color: black; } /* Class Selector - targets class="highlighted" */ .highlighted { background-color: yellow; } /* ID Selector - targets id="unique-element" */ #unique-element { font-weight: bold; }

4. The Cascade - Order Matters

When specificity is equal, the last rule wins:

.cascade-demo { color: purple; font-size: 16px; } .cascade-demo { color: orange; /* This wins - later in cascade */ font-weight: bold; }
This text shows how the cascade works. The color is orange (not purple) because it appears later in the CSS.

7. CSS Units - Getting Sizes Right

Choosing the right unit is crucial for professional-looking designs:

/* Absolute Units */ .pixel-text { font-size: 16px; } /* Fixed size */ /* Relative Units */ .rem-text { font-size: 1rem; } /* Relative to root font-size */ .em-text { font-size: 1.2em; } /* Relative to parent font-size */ .percent-width { width: 50%; } /* Relative to parent width */

8. Key Takeaways

  • CSS rules: selector { property: value; }
  • Specificity order: !important > ID > Class > Element
  • Cascade: When specificity is equal, last rule wins
  • Classes are best: Use classes for styling, IDs for JavaScript
  • Units matter: px for precision, rem for consistency, % for responsiveness
  • Avoid !important: Use it only when absolutely necessary
  • DevTools: Use browser inspector to see which rules are applied
